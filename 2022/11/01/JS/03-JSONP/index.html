同源策略

什么是同源策略？

为了避免被恶意攻击，浏览器端采用了 同源策略 来保证安全，它要求 IP地址、 **端口号 **和 协议 都相同的情况下，才可以发起请求并返回数据。

包括：

  • 协议：Http / Https
  • IP地址：一级域名？
  • 端口号：80、443……

那么，当，初学前端时，而又需要发起请求时，往往会产生 跨域 的报错信息，如何解决/避免跨域问题？

解决问题

  • 使用 JSONP （下文讲解）
  • 使用 webSocket（套接字）不了解
  • 使用 proxy ，服务器代理
  • 后端设置响应头

JSONP

jsonp 的解决方案是，使用 script 标签，通过 src 属性携带将要请求的地址；再由服务器调用，并返回响应的数据。
