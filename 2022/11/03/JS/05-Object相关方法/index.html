Object的相关方法

发表于2022-11-03|更新于2022-11-08

本文从 【实例方法】和【静态方法】两方面介绍Object的相关方法

Object的实例方法

已经废弃的方法：

  • __defineGetter__：定义get
  • __defineSetter__：定义set
  • __lookupSetter__：导出set
  • __lookupGetter__：导出get

重点：*

hasOwnProperty：*

判断对象上是否具备属性（不从原型上拿）

用法：【是否】具备此传入的【属性】

返回值：【布尔值】

参数：需要判断的【属性名】

【注意】 hasOwnProperty 不会读取【原型】上的属性！

isPrototypeOf：*

判断对象是否具备某对象的原型

用法：【是否】存在于传入的【对象】的原型链__proto__上

返回值：【布尔值】

参数：需要判断的【对象】

不那么重要：

propertyIsEnumerable

用法：【是否】能遍历传入的【属性】

返回值：【布尔值】

参数：需要判断的【属性】

所有的内置的属性都是false，所有的自定义属性返回true。

其他：

  • valueOf 提取对应的值

    toString 转为string字符串

  • tolocaleString 转为本地字符串

Object的实例属性

__proto__ 隐式原型

constructor 构造器

Object的静态方法

通过类名调用，只能被Object类名调用（也就是Object.什么什么）

assign（实现浅拷贝）

将后面的对象给到前面的对象 assign 返回一个新的对象 新的对象会和第一个对象指向同一个引用地址。

用法：将【追加对象】加给【目标对象】中之后，并返回这个【目标对象】

返回值：一个新的对象

参数：【目标对象】，【追加对象】

create

将一个对象创建到另一个对象的原型上

手写实现create

返回迭代器

keys values entries 返回的都是一个迭代器

is：解决 NaN 不等于 NaN

不那么重要的

一般是用于源码的设计，避免被增删改查

冻结 freeze（是否冻结 isFrozen） ===》 不能进行修改 （重新赋值无效，包含不可删除和拓展）

只能查询，不能增删改

密封 seal（是否密封 isSealed） ===》 不能删除（delete无效）也不能添加新的属性

能修改，不能增删

可拓展 preventExtensions（是否可拓展 isExtensible）===》 不能添加新的属性

能删改查，不能增加

重点！！！

setPrototypeOf

用法：原型设置

参数：（目标对象，原型对象）

getPrototypeOf

原型获取

getOwnPropertyDescriptor

用法：获取属性详情信息，返回属性对象（es内置）

返回值：返回【属性对象】

参数：（目标对象，目标属性）

【属性对象】

getOwnPropertyDescriptors

用法：获取 所有的 属性详情信息

返回值：返回【属性对象】

参数：（目标对象）

getOwnPropertyNames

用法：返回属性(不包含symbol)的数组

返回值：返回【属性数组】

参数：(目标对象)

getOwnPropertySymbols

用法：返回属性(只包含symbol)的数组

返回值：返回【symbol属性数组】

参数：(目标对象)

给对象添加属性

defineProperty

用法：定义一个属性

参数：（对象，属性名，属性对象）

属性对象基础属性

  • value 值
  • writable 是否可以设置
  • enumerable 是否可以遍历
  • configurable 是否可以删除

属性对象的访问器属性

  • get（） 表示获取 ===》 value
  • set（）表示设置 ===》 writable

defineProperties

定义多个属性、同上
